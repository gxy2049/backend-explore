# rust注意事项

1. Rust 是一种 预先编译（ahead-of-time compiled）语言
2. 静态类型语言，它必须在编译时就知道所有变量的类型
3. `./main # Windows 是 .\main.exe`

## 常用命令

1. `cargo check` 快速检查代码确保其可以编译，但并不产生可执行文件
2. `cargo build --release` 启用优化方式编译项目
3. 运行 cargo doc --open 命令会在本地构建所有依赖提供的文档，并在浏览器中打开

## 语法

1. 变量默认是不可变的，在变量名前使用 mut 来使一个变量可变
2. :: 语法表明 new 是 String 类型的一个 关联函数（associated function）
3. & 表示这个参数是一个 引用（reference），它允许多处代码访问同一处数据，而无需在内存中多次拷贝

## 概念

1. 库 crate 可以包含任意能被其他程序使用的代码，但不能独立执行
2. 语义化版本的几种表达?:
3. Cargo.lock 文件确保可重现构建
4. 变量遮蔽（shadowing）
5. 用户必须按下 enter 才能让 read_line 返回并提交他们的猜测，这会在字符串中附加一个换行符（newline）
6. continue，告诉程序进入 loop 的下一次迭代
7. 数据分配在栈（stack）上而不是堆（heap）上时

## 格式化打印

1. 由 `std::fmt` 中定义的一些列宏处理
   1. 只有实现了 fmt::Display 的类型才能用 `{}` 格式化
   2. format!
   3. print! 文本会打印在控制台 （io::stdout）
   4. println! 在末尾加换行符
   5. eprint! 文本会打印在标准错误输出（io::stderr）
   6. eprintln!

## 基本概念

1. 变量和可变性
   1. 默认不可变
   2. 与变量遮蔽之间的关系？？
      1. 实际上是在创建一个新变量
   3. 常量永远不可变，必须标注类型
      1. 把散落在应用程序中的硬编码值提取为常量
2. 数据类型
   1. rust是静态类型语言，必须在编译时就知道所有变量的类型。
   2. 标量
      1. 整形
         1. 长度、符号（正负）
      2. 浮点
      3. 布尔
         1. bool
      4. 字符
         1. 使用单引号来表示 char 字面值，而字符串字面值使用的是双引号
   3. 复合类型
      1. 元组（tuple）和数组（array）
      2. 元组
         1. 元组是一种将多个不同类型的值组合成一个复合类型的通用方式。元组长度固定：一旦声明，它的大小就不能增长或缩小。
         2. 如果一个表达式没有返回任何其他值，它就会隐式返回单元值。
         3. 可以解构、使用点号（`.`）后跟索引来访问
      3. 数组
         1. 数组中的每个元素都必须具有相同类型，长度是固定的。
         2. 数组越界
3. 函数
   1. 在函数签名中，必须 声明每个参数的类型。
   2. 函数可以把值返回给调用它的代码。我们不会给返回值命名，但必须在箭头（->）后面声明它的类型。
4. 注释
5. 语句和表达式
   1. 语句（Statements）是执行一些操作但不返回值的指令。
   2. 表达式（Expressions）计算并产生一个值。
      1. 函数调用是一个表达式。宏调用是一个表达式。
      2. 用大括号创建的一个新的块作用域也是一个表达式，
   3. 表达式的结尾没有分号。如果在表达式的结尾加上分号，它就变成了语句，而语句不会返回值。
6. 控制流
   1. if表达式
      1. 因为 if 是一个表达式，我们可以在 let 语句的右侧使用它
   2. 循环控制
      1. loop
         1. break
         2. continue
         3. 用于停止循环的 break 表达式后面加上想要返回的值；这个值会作为循环的返回值返回出来
         4. 循环标签
            1. 标签必须以单引号开头
      2. while
      3. for

## 所有权

1. 它使 Rust 无需垃圾回收器（garbage collector）也能提供内存安全保证
2. 所有程序都必须管理其运行时使用计算机内存的方式。一些语言中具有垃圾回收机制，在程序运行时有规律地寻找不再使用的内存；在另一些语言中，程序员必须亲自分配和释放内存。
   1. Rust 则选择了第三种方式：通过所有权系统管理内存，编译器在编译时会根据一系列的规则进行检查。如果违反了任何这些规则，程序都不能编译。在运行时，所有权系统的任何功能都不会减慢程序的运行。
3. 入栈比在堆上分配内存要快，因为（入栈时）分配器无需为存储新数据去搜索内存空间；其位置总是在栈顶。

### rust中的所有全

Rust 中的每一个值都有一个 所有者（owner）。
值在任一时刻有且只有一个所有者。
当所有者离开作用域，这个值将被丢弃，离开作用域时被移出栈。
